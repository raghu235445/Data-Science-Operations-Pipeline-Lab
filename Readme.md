## 🔍 Project Overview

Mobile money services such as MTN Mobile Money and Orange Money have transformed financial access across sub-Saharan Africa. However, their rapid adoption has been accompanied by a parallel surge in mobile money scams — a threat that disproportionately affects young, digitally active populations with limited fraud awareness.

This project applies a structured data science methodology to a real survey dataset of **800 respondents** collected in Cameroon. The goal is twofold:

1. **Descriptive** — Understand who gets scammed, how, and at what cost.
2. **Predictive** — Build machine learning models capable of identifying scam victims based on demographic and behavioural features.

The pipeline is implemented entirely in Python and follows industry-standard practices for data cleaning, encoding, scaling, dimensionality reduction, modelling, and evaluation. Results are documented in a professional Word report generated programmatically using the `docx` JavaScript library.

### Step 1 — Exploratory Data Analysis (EDA)

EDA is performed across six sub-steps, each producing a labelled console output and a saved plot.

#### 1a. Descriptive Statistics
Computed for all native numeric columns (`Age`) and additionally for `Amount_Lost_FCFA` and `Nb_Times` after numeric conversion:

| Metric | Age | Amount Lost (FCFA) |
|--------|-----|--------------------|
| Mean   | 29.1 | 26,317 |
| Median | 27.0 | 13,000 |
| Mode   | 25.0 | — |
| Std    | 9.04 | High (outlier-driven) |

The gap between the mean and median for `Amount_Lost_FCFA` signals a right-skewed distribution driven by a small number of very large losses.

#### 1b. Distribution Histograms
A histogram of `Age` reveals a **right-skewed distribution** — the majority of respondents are between 20 and 35 years old, with a long tail extending to 70. This confirms that the median is a more reliable central tendency measure than the mean for this column.

#### 1c. Class Imbalance Check
The target variable distribution:

| Class | Count | Percentage |
|-------|-------|------------|
| Not Victim (No) | 485 | 60.6% |
| Victim (Yes) | 315 | 39.4% |

The ~60/40 split is a moderate imbalance. While not extreme, it is sufficient to make plain accuracy misleading. All models are trained with `class_weight='balanced'` to compensate, and evaluation relies on **F1-Score** and **ROC-AUC** rather than accuracy.

#### 1d. Gender & Age Breakdown
- **Gender vs Victim:** Male respondents make up the majority of both groups, reflecting the larger male proportion in the survey sample. Female victimhood rates are notable relative to sample size.
- **Age by Victim Status:** Victims are most densely concentrated in the 22–35 age range, consistent with the most active mobile money user cohort.

#### 1e. Scam Methods Breakdown
The `Scam_Method` column contains semicolon-separated multi-label values. Each method is parsed and counted individually:

| Rank | Method | Count |
|------|--------|-------|
| 1 | Call from a stranger pretending to be an agent | 200 |
| 2 | Fake withdrawal or deposit message | 154 |
| 3 | False transaction of a customer | 57 |
| 4 | Reimbursement request for erroneous transfer | 33 |
| 5 | Other | 23 |

Phone-based social engineering dominates — underscoring the need for agent verification mechanisms and SMS authentication protocols.

#### 1f. Occupation vs Victim Status
A grouped bar chart reveals how victimhood varies across occupational categories, providing context for which professional groups are most exposed.

### Step 2 — Data Preprocessing

#### Dropped Columns
Columns excluded from modelling due to excessive sparsity, free-text content, or structural unsuitability:

| Column | Missing % | Reason Dropped |
|--------|-----------|----------------|
| `Timestamp` | 0% | Non-informative identifier |
| `Other_Method` | 95.8% | Almost entirely empty |
| `Reported_To` | 65.4% | High missingness + limited utility |
| `Suggestions` | 85.9% | Unstructured free text |
| `Info_Source` | 0% | Multi-label string; not parseable for modelling |

#### Target Encoding
`Victim` is binarised: `Yes → 1`, `No → 0`.

#### Categorical Encoding
All remaining `object`-type columns are encoded using `sklearn.preprocessing.LabelEncoder`. Missing values are filled with the string `'Unknown'` before encoding, preserving rows rather than dropping them.

Columns encoded include: `Social_Impact`, `Gender`, `Neighborhood`, `Occupation`, `MM_Usage_Duration`, `Heard_Scam`, `Scam_Method`, `Reported`, `Know_Detection`, `Need_Training`.

#### Imputation
`SimpleImputer(strategy='median')` is applied to handle any remaining NaN values in the numeric feature space. Median imputation is preferred over mean imputation because it is robust to the outliers observed in `Amount_Lost_FCFA` and `Nb_Times`.

#### Outlier Detection — Boxplots
Boxplots are generated for `Age`, `Amount_Lost_FCFA`, and `Nb_Times`. `Amount_Lost_FCFA` exhibits strong positive outliers — some respondents reported losses many times above the median. These are retained (not removed) as they represent genuine extreme events rather than data entry errors.

#### Correlation Heatmap
A lower-triangular correlation matrix is computed across all 13 model features plus the target. Key observations:
- `Scam_Method` and `Amount_Lost_FCFA` show the highest correlations with `Victim_Binary` — as expected, since these are direct indicators of a scam event.
- `Reported` shows moderate positive correlation with `Victim_Binary` — victims are more likely to have filed a report.
- Demographic features (`Age`, `Gender`) show weaker correlations, suggesting victimhood is not strongly age- or gender-determined when all other factors are considered.

---

### Step 3 — Feature Engineering & Scaling

#### StandardScaler
All 13 features are standardised using `sklearn.preprocessing.StandardScaler`:

```
z = (x − μ) / σ
```

This transforms each feature to zero mean and unit standard deviation. It is essential for **Logistic Regression** and **SVM**, which are sensitive to feature magnitude differences. It is applied consistently across all models for comparability.

#### Principal Component Analysis (PCA)
`PCA(n_components=5)` is applied to the scaled feature matrix to analyse the dimensionality structure of the data:

| Component | Variance Explained | Cumulative |
|-----------|--------------------|------------|
| PC1 | 24.4% | 24.4% |
| PC2 | 11.6% | 36.0% |
| PC3 | 10.1% | 46.1% |
| PC4 | 9.1% | 55.2% |
| PC5 | 8.8% | 64.0% |

Five components capture **64.0%** of total variance. Reaching the standard 80% threshold would require approximately 8 components. PC1's dominant share (24.4%) suggests a single primary pattern — likely the combined signal of scam experience variables (`Scam_Method`, `Amount_Lost_FCFA`, `Reported`).

> **Design Note:** PCA is used here as an analytical and visualisation tool. The original scaled features are used for modelling to preserve interpretability and feature importance analysis.

---

### Step 4 — Predictive Modelling

#### Train-Test Split
The data is split **80% training / 20% testing** using stratified sampling, preserving the 39.4%/60.6% class ratio in both partitions.

```
Training set: 640 samples
Test set:     160 samples
```

#### Models

| Model | Class | Key Parameters | Rationale |
|-------|-------|---------------|-----------|
| **Logistic Regression** | `sklearn.linear_model` | `max_iter=1000`, `class_weight='balanced'` | Linear baseline; highly interpretable; fast |
| **Random Forest** | `sklearn.ensemble` | `n_estimators=100`, `class_weight='balanced'` | Ensemble; handles non-linearity; provides feature importance |
| **SVM** | `sklearn.svm` | `kernel='rbf'` (default), `probability=True`, `class_weight='balanced'` | Kernel-based; powerful in well-scaled feature spaces |

All three models use `class_weight='balanced'` to automatically adjust sample weights inversely proportional to class frequency — compensating for the 60/40 imbalance without requiring data resampling.

---

### Step 5 — Evaluation

#### Metrics Used
- **F1-Score** — Harmonic mean of Precision and Recall; preferred over accuracy for imbalanced classes.
- **ROC-AUC** — Measures the model's ability to discriminate between classes at all classification thresholds.
- **Confusion Matrix** — Breaks down predictions into TP, TN, FP, FN for direct operational interpretation.
- **Classification Report** — Per-class Precision, Recall, and F1-Score.

#### Why Not Just Accuracy?
A naive classifier predicting "Not Victim" for every sample would achieve **60.6% accuracy** without learning anything. F1-Score and AUC penalise this behaviour and reward models that correctly identify actual victims (the minority class).

## 📈 Results Summary

All three models achieved near-identical and exceptionally strong performance:

| Model | Precision | Recall | F1-Score | ROC-AUC | Accuracy |
|-------|-----------|--------|----------|---------|----------|

| Random Forest | 0.98 | 0.98 | **0.976** | **0.998** | 98.1% |
| SVM | 0.98 | 0.97 | **0.976** | **0.999** | 98.1% |

The convergence of all three very different algorithms (linear, ensemble, kernel-based) to the same performance level provides strong confidence that the result reflects the structure of the data rather than an artefact of any particular model.

#### Feature Importance (Random Forest)
The top predictive features, ranked by mean decrease in Gini impurity:

1. `Scam_Method` — Direct indicator of scam exposure
2. `Amount_Lost_FCFA` — Direct indicator of financial loss
3. `Reported` — Victims are more likely to have reported
4. `Nb_Times` — Multiple incidents correlate with victimhood
5. `Know_Detection` — Awareness level is predictive
6. `Need_Training` — Desire for training correlates with victimhood
7. Demographic features (`Age`, `Gender`, etc.) — Lower importance

## 🔑 Key Findings

- **39.4% of 800 respondents (315 people) reported being scam victims** — a substantial prevalence rate.
- **Phone impersonation is the #1 scam vector** — 200 cases involved a stranger posing as a mobile money agent.
- **Fake SMS messages are the #2 vector** — 154 cases involved fraudulent withdrawal or deposit notifications.
- **Average loss: 26,317 FCFA (~USD 44)**, but the median of 13,000 FCFA reveals a skewed loss distribution.
- **Victims are concentrated in the 22–35 age range** — the core mobile money user demographic.
- **Scam_Method and Amount_Lost_FCFA are the strongest predictors** — behavioural/situational factors outweigh demographics.
- **All 3 models achieved F1 = 0.976 and AUC > 0.998** — victimhood is highly predictable from available survey features.
- **PCA: 5 components capture 64% of variance** — the dataset has moderate redundancy across features.

### Recommendations

- **Mobile operators** should implement real-time agent verification to reduce impersonation scams.
- **Public campaigns** should prioritise the 20–35 demographic and specifically address fake SMS and agent impersonation tactics.
- **The predictive model** could be deployed as a real-time fraud risk scoring tool, flagging high-risk transactions for review.
- **Future surveys** should reduce missingness in `Amount_Lost_FCFA` and `Nb_Times`, which exceeded 60% and limited modelling fidelity.

## ⚠️ Limitations & Future Work

### Limitations

| Area | Issue |
|------|-------|
| **Data source** | Self-reported survey data introduces recall bias and social desirability effects |
| **Missingness** | `Amount_Lost_FCFA` (66.5% missing) and `Nb_Times` (61.9% missing) significantly limit numeric analysis |
| **Encoding** | Label Encoding of nominal categoricals (e.g., `Scam_Method`) imposes an implicit ordinal relationship that may not exist |
| **Model performance** | Near-perfect scores may reflect dataset-specific patterns rather than fully generalisable real-world predictive power |
| **Generalisability** | The dataset covers specific Cameroonian neighbourhoods and may not generalise to other regions or populations |

### Future Work

- **One-Hot Encoding** for high-cardinality nominal features with a larger dataset to avoid artificial ordinality.
- **SMOTE** (Synthetic Minority Oversampling Technique) for datasets with more severe class imbalance.
- **IterativeImputer** (multivariate imputation) for a more principled treatment of the high-missingness numeric columns.
- **Cross-validation** (k-fold) instead of a single train-test split for more robust performance estimates.
- **Hyperparameter tuning** via `GridSearchCV` or `RandomizedSearchCV` for each model.
- **SHAP values** for model explainability — understanding *why* the model classifies a specific respondent as a victim.
- **External validation** on an independent dataset from a different survey period or geography.
